# financial-audit-system
财务审计系统 - 基于 Streamlit 开发的轻量级穿行测试审计工具

本人电脑软硬件有限，无法支持超200mb大数据，所以若是一张大表得拆分成公司，是按公司来做的

## 功能介绍
- 支持上传要抽样公司的样本数据，但表头只能有一行
- 支持上传历史数据，但表头最好不要太多，我测试是两行
- 支持 Excel 格式数据导入/导出
- 简洁直观的网页操作界面，可见运行和筛选过程

## Python 抽取规则
### 一、客户分层与抽样总量控制
1.  **客户分层**：以集团维度（含母公司及所有子公司）为统计范围，按客户订单数量占比划分客户层级：订单数量占比≥80%的客户定义为大客户，其余客户定义为小客户
2.  **抽样总量**：最终抽取样本总量需满足以下任一条件：
    - 与用户指定抽取量偏差在 ±10% 以内
    - 与过去两年实际抽样数量平均值偏差在 ±10% 以内
3.  **抽样配比**：在满足总量要求的前提下，大客户抽取数量必须多于小客户，确保对高价值客户的覆盖。

### 二、单据关联与筛选规则
1.  **单据匹配**：抽取样本必须同时包含出库单号与订单号。优先选择出库单号仅对应 1~2 个订单号的记录；若无法满足，则选择出库单号对应订单号数量最少的记录。
2.  **关联方排除**：抽取样本必须为非关联方交易，排除与本集团存在关联关系的客户记录。
3.  **金额有效性**：样本记录必须满足：
    - 交易数量 > 0
    - 未税总金额 > 0
4.  **特殊记账处理**：针对“一货三笔”记账场景（一笔负数出库、一笔本月暂估应收、一笔下月财务应收），仅抽取本月正数暂估应收记录。

### 三、时间分布与金额阈值
1.  **时间分散要求**：同一家公司下抽取的所有客户样本，其交易发生月份需均匀分散，避免集中在 1-2 个自然月。
2.  **金额阈值**：在满足上述所有规则的基础上，抽取样本的未税总金额必须大于 10,000 元。
